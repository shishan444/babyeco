## 版本说明

### 版本号格式

- 主版本号：重大功能变更或不兼容的 API 更改
- 次版本号：向下兼容的功能性新增
- 修订号：向下兼容的问题修复

### 发布流程

1. 开发阶段：功能开发和测试
2. SPEC 阶段：需求规格确认
3. 实现阶段：代码实现和验证
4. 同步阶段：文档更新和发布准备
5. 发布：版本号更新和上线
