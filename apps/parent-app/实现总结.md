## 架构决策

### 1. API 客户端模式
**决策:** 将 mock API 提取到单独的客户端模块

**理由:**
- 易于稍后替换为真实 API
- 集中式错误处理
- 一致的延迟模拟
- 可独立测试

### 2. react-hook-form 集成
**决策:** 使用带 Zod 解析器的 react-hook-form

**理由:**
- 性能优化（最小重渲染）
- 内置验证处理
- 优秀的 TypeScript 支持
- React 表单行业标准

### 3. 中间件路由保护
**决策:** 使用 Next.js 中间件而非 HOC

**理由:**
- 服务器端保护（更安全）
- 无客户端闪烁
- 自动重定向
- 更好的性能

### 4. 密码强度指示器
**决策:** 创建带视觉反馈的可重用组件

**理由:**
- 改善用户体验
- 减少支持请求
- 安全最佳实践
- 无障碍（WCAG 合规）
