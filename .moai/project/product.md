# BabyEco 产品文档

## 项目名称

**BabyEco** - 儿童行为激励经济系统

## 一句话描述

一套由家长制定规则、儿童自主参与的积分经济系统，用游戏化手段替代强制管教，培养儿童自律与目标管理能力。

---

## 目标用户画像

### 孩子端用户

| 属性 | 描述 |
|------|------|
| 年龄范围 | 6-12 岁小学阶段儿童 |
| 核心用户 | 9-12 岁（设计基准认知水平） |
| 兼容范围 | 6-8 岁（向下兼容） |
| 行为特征 | 需要外部激励建立良好习惯，对游戏化机制敏感，喜欢即时反馈 |
| 使用场景 | 完成家务、学习任务、阅读打卡、娱乐消费 |

### 家长端用户

| 属性 | 描述 |
|------|------|
| 身份 | 儿童监护人/家长 |
| 账号类型 | 单一主账号 |
| 核心诉求 | 希望用正向激励替代强制管教，培养孩子自律能力 |
| 使用场景 | 制定任务规则、审核任务完成、设置奖励兑换、监控孩子行为数据 |

---

## 核心价值主张

### 对家长的价值

1. **科学管教替代** - 用游戏化积分机制替代传统的强制命令和说教
2. **数据化洞察** - 通过任务完成率、积分流水等数据了解孩子行为模式
3. **灵活规则定制** - 根据家庭实际情况自定义任务类型和奖励机制
4. **健康监控** - 积分健康检测帮助家长及时发现行为失衡问题

### 对孩子的价值

1. **目标感培养** - 通过积分累积和愿望清单建立目标管理意识
2. **自主参与** - 主动选择任务和奖励，培养自主决策能力
3. **即时反馈** - 动效和音效提供成就感，强化正向行为
4. **财务启蒙** - 积分消费体验培养基础的"收支平衡"概念

## 主要功能模块概述

### 1. 任务系统

核心功能:
- 计次任务（每日可完成多次）
- 时间段任务（特定时段内完成）
- 家庭任务（家务劳动类）
- 负向扣分规则（不良行为扣分）
- 扣分保护规则（单次扣分上限限制）

### 2. 积分账户

核心功能:
- 积分余额实时显示
- 负债保护机制（负债上限 = 近 7 天实际赚取积分总量）
- 积分健康检测（扣分 > 赚分 50% 时预警）
- 定价参考模型（帮助家长合理设定积分价值）

### 3. 兑换中心

核心功能:
- 计时器功能（限时娱乐消费）
- 线下履约记录（实物奖励兑换跟踪）
- 愿望清单（目标累积与进度展示）
- 进度条可视化

### 4. 娱乐模块

核心功能:
- 电子书上传（家长上传内容）
- 双向积分配置（阅读赚积分 / 娱乐消费积分）
- 阅读学习模块

### 5. 成长看板

核心功能:
- 积分总览（收入/支出/余额）
- 当前目标进度
- 流水明细（时间线展示）

### 6. AI 问答

核心功能:
- 单角色 AI 助手
- 基础知识回答
- 情绪预警（检测负面情绪并通知家长）

### 7. 新手引导

核心功能:
- 5 步冷启动引导流程
- 快速建立初始任务和奖励体系

### 8. 家长端管理

核心功能:
- 审核中心（任务完成确认）
- 任务完成率监测
- 阅读报告推送

## 用户场景和使用案例

### 场景一：日常任务完成

```
1. 家长在家长端创建"整理房间"任务，设定奖励 10 积分
2. 孩子在孩端看到任务列表，完成后点击"打卡"
3. 家长收到审核通知，确认任务完成质量
4. 审核通过后，孩子获得 10 积分 + 动效反馈
```

### 场景二：愿望清单兑换

```
1. 孩子在孩端添加"去游乐园"到愿望清单，设定目标 500 积分
2. 孩子通过完成各类任务逐步累积积分
3. 达成目标后，孩子发起兑换申请
4. 家长确认后线下履约，系统记录兑换历史
```

### 场景三：阅读学习激励

```
1. 家长上传适龄电子书到娱乐模块，配置"阅读 30 分钟奖励 20 积分"
2. 孩子选择书籍进行阅读
3. 系统记录阅读时长，完成后自动发放积分
4. 家长端收到阅读报告推送
```

### 场景四：行为纠正（负向扣分）

```
1. 家长设定"不按时睡觉扣 5 积分"规则
2. 孩子发生该行为时，家长手动扣分
3. 系统检测扣分是否超过保护上限（当月预估可赚积分的 30%）
4. 若当月扣分 > 赚分 50%，系统自动推送家长提醒
```

## 成功指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 任务打卡完成率 | ≥ 80% | 已发布任务中实际完成的比例 |
| 家长审核响应率 | ≥ 70% | 孩子提交任务后家长及时审核的比例 |
| 孩子端 DAU/MAU | ≥ 40% | 用户活跃度指标 |
| 冷启动时长 | ≤ 10 分钟 | 新用户完成引导并开始使用核心功能的时间 |
| 积分健康比例 | 赚取 > 扣除 | 确保正向激励为主导 |

### 数据埋点需求

- 任务创建/完成/审核事件
- 积分变动事件（收入/支出）
- 页面访问和停留时长
- 功能模块使用频率
- 冷启动流程各步骤转化率

## UI 设计原则

### 设计基准

- 目标认知水平：9-12 岁
- 向下兼容：6-8 岁

### 交互规范

| 规范项 | 要求 |
|--------|------|
| 元素表达 | 图标与文字标签并存 |
| 按钮尺寸 | 最小 48×48dp，主操作按钮 60×60dp |
| 操作深度 | 每个界面核心操作不超过 3 步 |
| 反馈机制 | 大量即时动效与音效反馈 |

---

## 积分经济模型核心原则

1. **正向激励主导** - 孩子每月赚取积分必须显著多于被扣除积分
2. **扣分保护** - 单次扣分上限 ≤ 当月预估可赚积分的 30%
3. **负债控制** - 负债上限 = 近 7 天实际赚取积分总量
4. **健康预警** - 当月扣分 > 赚分 50% 时推送家长提醒
