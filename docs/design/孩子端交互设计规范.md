## 1. 设计原则

### 1.1 UI 设计基准

| 设计要素 | 规范 | 说明 |
|----------|------|------|
| **设计基准年龄** | 9-12岁为主，兼容6-8岁 | 6-8岁依赖图形和图标，9-12岁可理解数字和文字 |
| **元素表达** | 图标与文字标签并存 | 减少纯文字操作依赖，关键操作必须有图标辅助 |
| **按钮尺寸** | 最小48×48dp，主操作60×60dp | 符合移动端触控标准，低龄儿童手指点击准确度较低 |
| **操作深度** | 核心操作不超过3步 | 减少认知负担，防止孩子迷失在操作流程中 |
| **反馈机制** | 即时动效+音效反馈 | 完成任务、积分到账、称号解锁时必须有明确反馈 |
| **积分呈现** | 数字+可视化图形双重展示 | 数字满足9-12岁，图形兼容6-8岁 |

### 1.2 视觉风格

| 风格维度 | 具体要求 |
|----------|----------|
| **色彩** | 明亮温暖的配色，主色调建议使用黄色/橙色/绿色系，避免冷色 |
| **圆角** | 大圆角设计，卡片圆角16-24px，按钮圆角12-16px |
| **图标** | 线性或面性图标，风格统一，尺寸24-48px |
| **插画** | 可爱卡通风格，角色拟人化，增强亲和力 |
| **阴影** | 柔和投影，营造层次感，避免过于立体的质感 |

### 1.3 交互原则

1. **即时反馈** - 每次操作必须有视觉/听觉反馈
2. **正向激励** - 强调进步和成就，弱化失败和惩罚
3. **简单直接** - 减少选择项，提供清晰的引导
4. **容错设计** - 允许撤销，支持重试，不严厉批评

## 3. 全局组件规范

### 3.1 按钮组件

**主按钮**（Primary Button）

| 属性 | 规范 |
|------|------|
| 尺寸 | 高度60px，宽度自适应或全宽 |
| 圆角 | 16px |
| 背景 | 主色调填充 |
| 文字 | 白色，18-20px，粗体 |
| 图标 | 可选，尺寸24px，位于文字左侧 |
| 阴影 | 柔和投影，4px偏移 |
| 按下状态 | 缩放至95%，不透明度80% |

**次要按钮**（Secondary Button）

| 属性 | 规范 |
|------|------|
| 尺寸 | 高度48px，宽度自适应 |
| 圆角 | 12px |
| 背景 | 浅色背景或描边 |
| 文字 | 主色调，16-18px |

**图标按钮**（Icon Button）

| 属性 | 规范 |
|------|------|
| 尺寸 | 48×48dp 或 60×60dp |
| 图标 | 24-32px |
| 背景 | 圆形，可选背景色 |
| 触摸反馈 | 点击时缩放动画 |

**线性进度条**

| 属性 | 规范 |
|------|------|

| 属性 | 规范 |
|------|------|

**文本输入框**

| 属性 | 规范 |
|------|------|

| 聚焦状态 | 边框变为主色调，显示阴影 |
| 占位符 | 提示性文字，避免术语 |

## 5. 状态设计

### 5.1 空状态

| 场景 | 视觉表现 | 引导文案 |

| 场景 | 表现方式 |
|------|----------|

| 列表分页 | 底部加载指示器 + "加载更多中..." |

### 5.3 成功状态

| 场景 | 反馈方式 |
|------|----------|
| 任务打卡成功 | 全屏庆祝动效（彩带/星星） + 音效 + "积分+10"浮动提示 |

## 7. 动效规范

### 7.1 动效触发时机

| 触发场景 | 动效类型 | 说明 |
|----------|----------|------|
| 任务打卡成功 | 庆祝动效 | 彩带/星星从顶部落下，1-2秒 |
| 积分到账 | 数字跳动 | 积分数字从旧值滚动到新值 |
| 兑换成功 | 对勾动画 | 对勾从中心放大弹出 |
| 等级提升 | 徽章展示 | 徽章从下方升起 + 光效 |
| 愿望达成 | 礼花动效 | 全屏礼花爆炸效果 |
| 页面切换 | 滑动过渡 | 新页面从右侧滑入 |
| 下拉刷新 | 旋转动画 | 顶部图标旋转 |
| 加载中 | 脉冲动画 | 元素缩放脉冲 |

### 7.2 动效描述规范

**示例：任务打卡成功动效**

> **触发时机**：用户点击"我完成了"按钮，打卡提交成功后
> **动效类型**：全屏庆祝动效
> **具体表现**：
> - 彩色纸屑从屏幕顶部随机位置落下
> - 持续时长约1.5秒
> - 同时播放"叮~"的成功音效
> - 屏幕中央显示"太棒了！+10积分"浮动文字
> - 动效结束后自动返回任务列表

**示例：积分数字跳动**

> **触发时机**：积分数值发生变化时
> **动效类型**：数字滚动动画
> **具体表现**：
> - 数字从旧值快速滚动到新值
> - 持续时长约0.5秒
> - 滚动过程中数字略放大（1.1倍）
> - 结束时恢复原尺寸并播放音效

### 7.3 音效规范

| 场景 | 音效类型 | 说明 |
|------|----------|------|
| 按钮点击 | 轻轻"哒"声 | 短促，低音量 |
| 打卡成功 | 清脆"叮"声 | 明亮，愉悦 |
| 积分到账 | 钱币"叮当"声 | 金属质感 |
| 兑换成功 | 胜利号角 | 欢快 |
| 等级提升 | 魔法音效 | 梦幻 |
| 错误提示 | 温和"咚"声 | 不刺耳 |
| 等待完成 | 提示音 | 3声渐进 |

## 8. 文案规范

### 8.1 语言风格

- **语气**：温暖、鼓励、友好，像朋友一样
- **称呼**：使用"你"而非"您"，拉近距离
- **用词**：简单易懂，避免专业术语和复杂词汇
- **标点**：适当使用感叹号表达热情，但不滥用

### 8.2 禁用词汇

| 禁用 | 替代 |
|------|------|
| 惩罚 | 扣分 |
| 失败 | 再试试 |
| 错误 | 不对哦 |
| 不行 | 暂时不可以 |
| 必须 | 可以试试 |

### 8.3 按钮文案

| 场景 | 文案 |
|------|------|
| 主操作 | 使用动词，如"开始阅读"、"提交打卡" |
| 次要操作 | 使用"查看详情"、"稍后再说" |
| 取消操作 | 使用"返回"、"取消"、"先不啦" |
| 确认操作 | 使用"好的"、"确定"、"就这样" |

### 8.4 提示文案

| 场景 | 文案示例 |
|------|----------|
| 成功 | "太棒了！"、"你真厉害！"、"完成啦！" |
| 等待 | "等一下哦..."、"马上就好..." |
| 错误 | "哎呀，出错了"、"再试一次吧" |
| 鼓励 | "加油！"、"你可以的！"、"继续努力！" |
