## 1. 设计原则

### 1.1 设计理念

| 设计要素 | 规范 | 说明 |
|----------|------|------|
| **设计定位** | 高效、不打扰 | 家长使用时间有限，快速完成任务是核心 |
| **信息密度** | 适中 | 比孩子端高，比管理后台低 |
| **操作效率** | 核心操作不超过3步 | 减少操作路径，提高效率 |
| **通知策略** | 精准推送 | 避免通知疲劳，只推送重要事项 |
| **视觉风格** | 简洁现代、专业可信 | 与孩子端区分，建立产品专业形象 |

### 1.2 视觉风格

| 风格维度 | 具体要求 |
|----------|----------|
| **色彩** | 成熟稳重的配色，主色调建议使用蓝色/绿色系，可少量使用暖色作为点缀 |
| **圆角** | 适中圆角，卡片圆角8-12px，按钮圆角6-8px |
| **图标** | 线性图标为主，尺寸16-24px |
| **阴影** | 轻微投影，避免过强的层次感 |
| **排版** | 信息层级清晰，使用字重和大小区分重要性 |

### 1.3 交互原则

1. **效率优先** - 常用操作一键直达，减少层级
2. **信息透明** - 关键数据一目了然，支持钻取查看详情
3. **容错友好** - 重要操作二次确认，支持撤销
4. **多孩支持** - 快速切换孩子档案，数据独立隔离

## 3. 全局组件规范

### 3.1 按钮组件

**主按钮**（Primary Button）

| 属性 | 规范 |
|------|------|
| 尺寸 | 高度44px，宽度自适应或全宽 |
| 圆角 | 8px |
| 背景 | 主色调填充 |
| 文字 | 白色，16px，中粗 |
| 图标 | 可选，尺寸18px，位于文字左侧 |
| 阴影 | 轻微投影，2px偏移 |

**次要按钮**（Secondary Button）

| 属性 | 规范 |
|------|------|
| 尺寸 | 高度40px，宽度自适应 |
| 圆角 | 6px |
| 背景 | 浅色背景或描边 |
| 文字 | 主色调，14-16px |

**文本按钮**（Text Button）

| 属性 | 规范 |
|------|------|
| 尺寸 | 高度32px |
| 背景 | 透明 |
| 文字 | 主色调或灰色，14px |

### 3.2 卡片组件

**数据卡片**（Data Card）

```
┌─────────────────────────────────────┐
```

| 属性 | 规范 |

| 背景 | 白色，轻微投影 |
| 内边距 | 12-16px |
| 间距 | 卡片之间8-12px |

### 3.3 表单组件

**输入框**

| 属性 | 规范 |
|------|------|
| 高度 | 44-48px |

| 标签 | 顶部浮动标签或左侧固定标签 |

| 属性 | 规范 |

| 属性 | 规范 |
|------|------|

| 开启状态 | 主色调填充 |
| 关闭状态 | 灰色 |
| 动画 | 滑块平滑过渡 |

**分段控制器**（Segmented Control）

| 属性 | 规范 |

| 背景 | 浅灰色 |

## 5. 状态设计

### 5.1 空状态

| 场景 | 视觉表现 | 引导文案 |
|------|----------|----------|
| 任务列表为空 | 插画 + "还没有任务" | "点击上方创建第一个任务" |
| 审核列表为空 | 对勾插画 + "全部处理完成" | "太棒了！没有待处理事项" |
| 报告列表为空 | 图表插画 + "暂无报告" | "积累更多数据后会生成报告" |
| 内容列表为空 | 书本插画 + "还没有内容" | "点击上传内容开始添加" |

### 5.2 加载状态

| 场景 | 表现方式 |
|------|----------|
| 页面加载 | 骨架屏，灰色块占位 |
| 数据刷新 | 顶部下拉刷新动画 |
| 表单提交 | 按钮变为加载状态 + 保存中... |
| 图表加载 | 图表区域显示加载动画 |

### 5.3 成功状态

| 场景 | 反馈方式 |
|------|----------|
| 创建任务成功 | Toast提示 + 返回列表 |
| 审核通过 | Toast提示 + 卡片移除 |
| 保存设置成功 | Toast提示 + "已保存" |
| 上传内容成功 | Toast提示 + 刷新列表 |

### 5.4 错误状态

| 场景 | 视觉表现 | 操作引导 |
|------|----------|----------|
| 网络错误 | 断网提示 | [刷新] |
| 表单验证失败 | 字段标红 + 错误提示 | 修正后重试 |
| 操作失败 | 错误提示 | [重试] |
| 权限不足 | 锁定图标 + 说明 | [升级账号] |

### 5.5 警告状态

| 场景 | 视觉表现 | 引导文案 |
|------|----------|----------|
| 积分健康预警 | 橙色提示卡片 | "本月扣分已接近赚分的50%" |
| 定价偏高 | 黄色警告 | "建议调低积分要求或增加任务" |
| 完成率过低 | 蓝色提示 | "任务完成率<50%，建议降低难度" |

## 7. 文案规范

### 7.1 语言风格

- **语气**：专业、简洁、友好
- **称呼**：使用"您"，表示尊重
- **用词**：准确、明确，避免歧义
- **句式**：简洁为主，一句表达一个意思

### 7.2 按钮文案

| 场景 | 文案 |
|------|------|
| 主要操作 | 动词开头，如"创建任务"、"保存" |
| 次要操作 | "取消"、"返回"、"跳过" |
| 确认操作 | "确认"、"确定"、"同意" |
| 删除操作 | "删除"（需二次确认） |

### 7.3 提示文案

| 场景 | 文案示例 |
|------|----------|
| 成功 | "保存成功"、"操作完成" |
| 错误 | "操作失败，请重试" |
| 警告 | "确定要删除吗？" |
| 引导 | "点击创建第一个任务" |
| 空状态 | "暂无数据" |

### 7.4 标签文案

| 标签类型 | 文案 |
|----------|------|
| 任务类型 | "每日任务"、"每周任务"、"家庭任务" |
| 状态 | "待审核"、"已通过"、"已驳回"、"已完成" |
| 时间 | "今天"、"本周"、"本月" |
| 优先级 | "高"、"中"、"低" |
